Classpublic class CategoryAxis
InheritanceCategoryAxis AxisBase mx.core.UIComponent

CategoryAxis is the class which displays category axis for the Serial Chart. The chart can have only one category axis. If you do not add category axis to the Serial chart, it will be created automatically.

autoTruncateproperty
autoTruncate:Boolean  [read-write]

Specifies if category labels should be truncated if they don't fit. Doesn't work if labels are rotated, or category axis is vertical.

The default value is false.

dateFormatsproperty 
dateFormats:Array  [read-write]

Array of date format objects. Date format object must have "period" and "format" items. Available periods are: fff - millisecond, ss - second, mm - minute, hh - hour, DD - date, MM - month, YYYY - year. format can contain any string supported by mx.formatters.DateFormatter class. For example: {period:"DD", format:"MMM DD"}

The default value is [{period:'fff',format:'JJ:NN:SS'},{period:'ss',format:'JJ:NN:SS'},{period:'mm',format:'JJ:NN'},{period:'hh',format:'JJ:NN'},{period:'DD',format:'MMM DD'},{period:'MM',format:'MMM'},{period:'YYYY',format:'YYYY'}].

Implementation

equalSpacingproperty 
equalSpacing:Boolean  [read-write]

In case your category axis values are Date objects and parseDates is set to true, the chart will parse dates and will place your data points at irregular intervals. However if you want dates to be parsed (displayed on the axis, baloons, etc), but data points to be placed at equal intervals (omiting dates with no data), set equalSpacing to true.

The default value is false.

minPeriodproperty 
minPeriod:String  [read-write]

Specifies the shortest period of your data. This should be set only if parseDates is set to "true".

  • fff - millisecond
  • ss - second
  • mm - minute
  • hh - hour
  • DD - day
  • MM - month
  • YYYY - year

    • The default value is DD.

Installing components

To use SWC files in your Flex projects, you should add them to the project's library path.

  1. With a project selected in the Flex Navigator view, select Project > Properties > Flex Build Path.
  2. Click on the Library Path tab.
  3. Click on the Add Swc button.
  4. Browse to and select the amChartsFB3.swc file if you work with Flex Builder 3. Select amChartsFB4.swc if you work with Flash Builder 4.
  5. Click OK
  6. The component is now added. You should see amCharts folder in the components window (only available when you are in design mode).
Visit Flex documentation for more options on installing components.

1.7.0.0

AutoMargins
Since v 1.7.0.0 our flex charting components has a long awaited feature - automatic calculation of margins. This means you no longer need to worry that axes labels might not fit into the available space.

autoMargins property was added to AmRectangularChart class, with default value set to true. This means that if you update swc file with a new one, the look of your charts might change! If you want to keep things old-style, set this property to false.

If autoMargins is set to true, then margins where axes (value or category) will be adjusted for the values to fit. Other margins won't be changed. Margins also won't be changed if values are inside plot area (inside is set to "true" for the axis). You can even tell the chart to ignore one or another axis when calculating margins. Set ignoreAxisWidth to true for this axis (it's a property of AxisBase).

When margins are updated, chart dispatches marginsUpdated event (AmChartEvent). This means you can monitor when margins changed and adjust some other layout you need.

Background color and opacity for axis labels
labelBackgroundColor and labelBackgroundAlpha styles added to AxisBase class. As you guessed, they allow to display a rectangle below axis labels. It might be useful when labels are inside plot area.

Maximum length of axis label
maxLabelLength property added to AxisBase class. Most likely you'll use it with category axis. If your category values are long and you want to truncate them, set maxLabelLength to 10 or so. Balloon with full label will be displayed on mouse over.

filter for bullet or column
filter property added to GraphDataItem Class. You can set BitMapFilter for individual column/bullet using this property.

Find out how many data points a graph has
A new property, dataPointCount was added to AmGraph class. It will return a number of data points this graph has.

Fly-in direction
startFrom style added to AmCoordinateChart with possible values "top", "bottom", "left" and "right". It specifies where from the graphs will fly-in. This affects line, smoothed line and step graphs only.

Private to Protected
We changed most of properties and methods from private to protected. If you have sources of our components this will allow extending our components more easily.

1.6.1.0

A possibility to use prefixes for big and small number added. For example, instead of 1000 you can show 1k, instead of 2000000 - 2M, instead of 0.000001 - 1μ, etc. This works for value axes and for legend. To enable this feature, you should set usePrefixes="true" on the chart, or explicitly on the valueAxis. You can also control prefixes. Two properties, prefixesOfBigNumbers and prefixesOfSmallNumbers added. Default value of prefixesOfBigNumbers is:

[{number:1e+3,prefix:"k"},{number:1e+6,prefix:"M"},{number:1e+9,prefix:"G"},{number:1e+12,prefix:"T"},{number:1e+15,prefix:"P"},{number:1e+18,prefix:"E"},{number:1e+21,prefix:"Z"},{number:1e+24,prefix:"Y"}]

Default value of prefixesOfSmallNumbers is:

[{number:1e-24, prefix:"y"},{number:1e-21, prefix:"z"},{number:1e-18, prefix:"a"},{number:1e-15, prefix:"f"},{number:1e-12, prefix:"p"},{number:1e-9, prefix:"n"},{number:1e-6, prefix:"?"},{number:1e-3, prefix:"m"}]

You can change them if you want. For example, if you don't want 1000 to be replaced with 1k, delete first object from default value of prefixesOfBigNumbers array.

a new property, gridPosition with optional values start and middle added to CategoryAxis. It works only if you do not parseDates. Default value is middle. Set it to start if you want grid lines to be placed in the beginning of a cell.

Min/max calculation algorithm when axis uses logarithmic scale improved. We optimized it and we think it's perfect now. Or close to.

Scrollbars category axis will use the same date format as main category axis of a chart. We also added a possibility to get categoryAxis of a scrollbar.

When graph.connect was set to false, radar chart still drawn a line to the center of the radar, this is fixed now.

1.6.0.0

This is important update!

Some changes which might affect previous behavior were made in this version, read the following carefully.

AmCharts Bundle (FB4 build) now can be used with HERO SDK. This means you can build mobile applications with amCharts for Android and iPhone now. To accomplish this we had to get rid of classes, not supported by HERO SDK. We made changes both in FB3 and FB4 build so that it would be easier to maintain and document. This also made our components much lighter and a bit faster. Below listed changes which might have influence to your applications with previous versions of amCharts.

AmLegend Class no longer extends Canvas, it extends UIComponent instead. This means that AmLegend doesn't support styles/properties/methods specific to Canvas anymore. It still has styles backgroundColor, backgroundAlpha, borderColor and borderAlpha. This change might cause error on your application if you used Canvas specific styles or properties. If you need them anyway, you can always put our legend into Canvas container.

We no longer use LinkButton for zoomOutButton (property of AmRectangularChart). This property can accept any object. This means, if you used LinkButton before and still want to use it, you can do this and the chart will work fine. We build our own AmSimpleButton class and use it for this button. You can also use it if you want to customize it a bit. We also added zoomOutText property to AmRectangularChart, so that you wouldn't need to create an instance of a button if you just want to change the text of the button.

New features in this version:

dateToCoordinate(date) method was added to CategoryAxis. It allows finding x (or y if the chart is rotated) coordinate of the date. Zero coordinate is at the beginning of the axis, so you might need to add marginLeft to get right position.

categoryToCoordinate(category) method was added to CategoryAxis. It does the same as dateToCoordinate method only is made when parseDates is false.

showBullet style added to AmBalloon class. When cornerRadius style of a balloon is >0, small round bullet is shown near the balloon. If you don't want this bullet, set this style to false.

"down" option added to pointerOrientation style of AmBalloon. Use it when building applications for touch screen devices.

selectWithoutZooming property added to ChartCursor. If you set it to true, then you will be able to mark some chart's area and the chart won't zoom in. zoomable property should still be set to true in order this to work.

columnsAboveLines property added to AmSerialChart. It allows specify whether column graphs should be in front of or behind the line/area graphs. By default columns are behind lines.

Bug fixes:

There was no way to change legend's data provider at runtime.

1.4.0.0 beta

Upgrading from previous version

The new chart types caused some changes in the structure, so there will be several things you'll need to do when migrating from previous versions. We are sorry for this inconvenience but we feel it is the absolutely necessary change for the sake of consistency in all future versions.
  1. AmCartesianChart class renamed to AmSerialChart - simply change the name in your applications.
  2. "cartesian" folder with classes like AmGraph, ChartScrollbar, ChartCursor changed to "chartClasses" - if you import one of these classes in your Actionscript, you should change the path to the class.
  3. AmChart, AmBalloon, AmLegend classes were moved from com.amcharts to com.amcharts.chartClasses. Change the paths to these classes too.
  4. Even if you do not use these classes in your ActionScript, but only mxml, Flex builder can cache the old paths to these classes and show errors when compiling application. The most easy way to solve this is to remove all your amChart objects from the application, save it and bring it back exactly as it was. Now the application should compile properly.
  5. CartesianChartEvent is renamed to SerialChartEvent.
  6. PieChartEvent and SerialChartEvent (previously CartesianChartEvent) do not have dataUpdated event type anymore. dataUpdated event is now AmChartEvent's type.

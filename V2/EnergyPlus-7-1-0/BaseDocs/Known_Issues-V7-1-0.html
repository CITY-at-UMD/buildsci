Folder: EnergyPlus
CR Number: 7,872
Type: Defect
Addressed In:
Synopsis for Users: Exterior shading surfaces modeled using "FullInteriorAndExteriorWithReflections" can show some solar getting through. When "*WithReflections" is active a partially sunlit shading surface reflects uniformly from the entire surface. This is as designed.
Further Details for Users:
Work Around: If using WithReflections, shading surfaces should be brokent into multiple surfaces at lines of intersection with another shading surface. This also includes places where another surface may tee into a shading surface. For example, a building is shaded by surface A, 20m long by 6 m high, running North-South. Shading surface B, running East-West, connects to surface A. Surface A should be broken into two parts at the point where Surface B connects to it.


Folder: EnergyPlus
CR Number: 8,123
Type: Defect
Addressed In:
Synopsis for Users: SimHVAC: Max Iteration errors when HX used in OA system and Controller input set to BypassWhenOAFlowGreaterThanMinimum.
Further Details for Users: This specific problem is not directly related to the HX. The supply air temperature appears to be in a floating condition. When the VAV dampers respond to the air flow to meet the zone load, the supply air flow rate is affected which changes the mixed air temperature. This directly affects the supply air temperature and the VAV dampers again try to adjust to meet the zone load.
Work Around: No specific solution at this time. The remedy may be related to holding the supply air temperature at some fixed value (i.e., heating and cooling coils active at all times and maintaining the same supply air temperature set point).

Folder: EnergyPlus
CR Number: 8,650
Type: Defect
Addressed In:
Synopsis for Users: The plant solver sometimes has problems detecting when it is sufficiently converged and needs to be improved. When modeling complex plant systems, care should be taken to ensure that the solution is converged.
Further Details for Users: The ConvergenceLimits objects now contains fields for minimum and maximum plant sub iterations. This is especially important for complex plant systems which include those with inter-connected loops, common pipes, heat recovery, and on-site generators.Convergence of plant models can be investigated by varying the value for the minimum plant iterations and examing how results change. Increase the minimum and maximum limits until the change in results is small for your purposes.
Work Around:

Folder: ExpandObjects, Type: Defect (4 item(s))
Folder: ExpandObjects
CR Number: 8,066
Type: Defect
Addressed In:
Synopsis for Users: HVACTemplate:System:VAV and PackagedVAV central heating coil can be wrong (too large or too small), because HVACTemplate currently writes a fixed value of 0.3 for Sizing:System "Minimum System Air Flow Ratio".
Further Details for Users: The value for "Minimum System Air Flow Ratio" in the Sizing:System object is used to establish the flow rate used for sizing a central heating coil in a VAV system. EnergyPlus requires a value to be input here, it is not currently autosizable. In HVACTemplate, when the expanded if file is written, there is not enough information available to effectively calculate a better value. The user may need to edit the expidf file to adjust this value for proper heating coil sizing.
Work Around: Edit the expidf file and adjust the value for Siziing:System, Minimum System Air Flow Ratio.
